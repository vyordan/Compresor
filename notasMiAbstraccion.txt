Entrada: std::vector<uint8_t> con los datos ya leídos
Salida: std::vector<uint8_t> con los datos procesados
Errores: Excepciones
Header: Primer byte identifica el algoritmo
Métodos: Solo dos, compress y decompress
Eso es todo lo que necesita ICompressor (este es el nucleo, solo se encarga de procesar un bufer), suficiente para que CLI, GUI y gRPC lo consuman sin saber qué hay adentro.

Interfaz lógica de ICompressor
Contrato:

Recibe un std::vector<uint8_t> con los datos ya leídos por el caller
Devuelve un std::vector<uint8_t> con los datos procesados
Ante cualquier error lanza una excepción, el caller usa try catch
El buffer comprimido siempre lleva un header con al menos un byte que identifica el algoritmo usado

Métodos:
compress — Recibe el buffer original, devuelve el buffer comprimido con header incluido
decompress — Recibe el buffer comprimido, lee el header para identificar el algoritmo, devuelve los datos originales
Identificadores de algoritmo en el header:
1 → RLE
2 → Huffman
3 → LZ77
4 → LZ78
5 → LZW
6 → Variante propia
